# Formatting Rules

Короткий reference для оформления в IWE-репозиториях.

## Базовые правила

- Пиши короткими абзацами и плоскими списками.
- Один список = один уровень. Вложенные списки не использовать.
- В таблицах сначала факт, потом интерпретация.
- Название РП = артефакт или проверяемый результат, а не намерение.

## Для стратегии и клиентских артефактов

- `WeekPlan` и `DayPlan` ведутся как операционные таблицы статусов.
- `WP-context` фиксирует критерий готовности, источник и секцию «Осталось».
- Клиентские артефакты (`docs/`, презентации, стенограммы, memo) оформляй так, чтобы по ним можно было восстановить решение без чата.

## Где лежит источник истины

- Формат протоколов и ритуалов: `memory/protocol-*.md`
- Жёсткие различения: `.claude/rules/distinctions.md` и `memory/hard-distinctions.md`
- Репозиторные нюансы: `repo/CLAUDE.md`
