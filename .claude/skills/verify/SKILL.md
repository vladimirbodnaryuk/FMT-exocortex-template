# Верификация артефакта

> **Роль:** VR.R.001 Верификатор (PACK-verification)
> **Принцип:** Context isolation (VR.SOTA.002) — проверяю результат по эталону, НЕ процесс создания.

Артефакт для проверки: $ARGUMENTS

## Шаг 1. Определить эталон

Какой документ является reference для этого артефакта?

| Тип артефакта | Эталон |
|---------------|--------|
| Pack-сущность | SPF pack-template + доменные принципы Pack |
| Описание метода | SPF process/07 + Pack |
| Код (DS) | CLAUDE.md репо + Pack-описания сервисов |
| Публикация/пост | Контент-стратегия + Pack + ЦА |
| Архитектурное решение | DP.ARCH.001 §7 (→ используй /archgate вместо /verify) |
| План (WeekPlan/DayPlan) | Протоколы Open/Close |

Если эталон не определяется → **СТОП.** Сообщи: «Эталон не найден. Это валидация (экспертная оценка), не верификация. Нужен рецензент, не верификатор.»

## Шаг 2. Загрузить контекст

**Загрузить:**
1. Артефакт (проверяемый файл)
2. Эталон (reference-документ из Pack/SPF)
3. Чеклист (если есть для этого типа артефакта)

**НЕ загружать:**
- Задание создателя (что просили сделать)
- Промежуточные рассуждения
- Историю итераций

## Шаг 3. Проверка

### 3a. Формальная (VR.VT.001)
- [ ] Frontmatter корректен (id, name, kind, status, created)
- [ ] Структура соответствует шаблону
- [ ] Ссылки валидны
- [ ] ID уникальны

### 3b. Содержательная (VR.VT.002)
- [ ] Утверждения соответствуют принципам домена (Pack)
- [ ] Терминология = UL (Ubiquitous Language)
- [ ] Нет подмен (VR.D.001–004)
- [ ] Полнота: все аспекты покрыты
- [ ] Непротиворечивость с существующими сущностями

### 3c. Кросс-контекстная (VR.VT.003, если применимо)
- [ ] Согласованность с связанными контекстами
- [ ] Нет orphans/ghosts/drift/stale

## Шаг 4. Verdict

Вывести в формате:

```
## Verdict

**Результат:** pass / fail / conditional
**Артефакт:** [что проверяли]
**Эталон:** [по чему проверяли]
**Тип верификации:** formal / content / cross-context
**Верификатор:** VR.R.001 (модель: [текущая])

### Несоответствия

| # | Severity | Описание | Эталон говорит | Артефакт содержит |
|---|----------|----------|---------------|-------------------|
| NC-001 | critical/major/minor | ... | "..." | "..." |

### Резюме

[1-2 предложения]
```

**Критерии:**
- **pass:** 0 несоответствий или только minor
- **conditional:** ≥1 major, 0 critical
- **fail:** ≥1 critical
