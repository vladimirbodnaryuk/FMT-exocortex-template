# АрхГейт — оценка архитектурного решения

Выполни оценку решения: $ARGUMENTS

## Шаг 1. Принципы (ДО оценки)

Сверь решение с принципами 2-го уровня (DP.ARCH.001 §7). Если решение нарушает принцип — сообщи и предложи исправление до подачи на оценку.

## Шаг 2. Оценка ЭМОГССБ

Оцени решение по 7 характеристикам (шкала 1-10):

| Характеристика | Вопрос | Оценка |
|----------------|--------|--------|
| **Э**волюционируемость | Что сломается при изменении? | ? |
| **М**асштабируемость | Что будет при 10x? | ? |
| **О**бучаемость | Сколько читать, чтобы начать? Усиливает мышление или заменяет (экзоскелет vs протез)? | ? |
| **Г**енеративность | Создаёт платформу? Работает в шаблоне экзокортекса? | ? |
| **С**корость | Бот <3 сек, CLI <1 сек? | ? |
| **С**овременность | Проверь `memory/sota-reference.md` | ? |
| **Б**езопасность | Какие угрозы? PII, секреты, injection surface? Может ли пользователь забрать данные и уйти? Создаёт ли lock-in? | ? |

### Coordination cost check (при мультиагентных/мультисистемных решениях)

Coordination overhead < parallelism gain? Три условия: (1) context isolation, (2) parallelism gain, (3) tool specialization. Все три НЕ выполнены → single-agent.

## Шаг 3. Вердикт L1

- **Итого:** X/70
- **Порог:** ≥56 (≥8 в среднем)
- Характеристика ≤7 → проверь покрытие принципами (DP.ARCH.001 §7.1). Принципа нет → предложи.

Если оценка <56 → «Решение НЕ проходит АрхГейт. Слабые характеристики: [список]. Рекомендация: доработать.» СТОП.

## Шаг 4. Доменные расширения (L2)

> Полное описание: DP.M.005 §9. L2 = информативный (не блокирующий).

**4a. Определи триггеры:** Какие компоненты архитектуры затрагивает решение?

| Триггер | L2-характеристика |
|---------|-------------------|
| Формат хранения user-data, схема данных, зависимость от вендора/API | **Переносимость данных** (L2.1) |
| ИИ-система (Зона А), Intervention Loop, недетерминированный компонент | **Наблюдаемость** (L2.2) |
| ИИ-оценка пользователя (квалификация, прогресс, рекомендация, обратная связь) | **Объяснимость** (L2.3) |
| Оценка/измерение (ЦД, метрики, stage, score) | **Воспроизводимость** (L2.4) |
| Автоматическое действие от имени пользователя | **Контролируемость** (L2.5) |
| Миграция данных, хранилище, схема ЦД | **Сохранность знаний** (L2.6) |
| Внешняя интеграция, протокол обмена, MCP Registry | **Интероперабельность** (L2.7) |

Ни один триггер не сработал → пропусти L2, перейди к шагу 5.
