## Специфика: R4 Автор — презентации (МИМ)

> Tseren — спикер МИМ, автор контента для нескольких аудиторий.

**Стиль:** короткие слайды, минимум текста. Структура: крючок → боль → инструмент → ограничение → глубина.

**Три сегмента:** A (новички), B (практики), C (эксперты). Сигнальные фразы маркируют адресата.

**Порядок:** A — простой пример, без терминов → B — кейс → чек-лист → шаблон → C — намёк на глубину, не раскрывать.

**Структура семинара:** 0. Крючок (всем) → 1. Боль (A+B) → 2. Диагностика (A+B) → 3. Пример (A) → 4. Мини-инструмент (B) → 5. Ограничение (B→C) → 6. Намёк на глубину (C) → 7. Куда дальше (всем).

**Переходы:** через ловушку: *«Это работает. Но вот где ловушка...»*

**Вопросы:** от A → просто, не углубляться. От C → «хороший вопрос для отдельного разбора» + 1 строчка глубины. Не тянуть аудиторию на уровень вопрошающего.

**Главное правило:** Глубина не раскрывается — она приоткрывается.
