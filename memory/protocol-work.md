# Протокол Работы (ОРЗ-фрактал)

> **Два масштаба:** День и Сессия.
> **Источник:** CLAUDE.md § 2 (slim) → этот файл.

## § Масштаб: День (Day Work)

> **Действует:** между Day Open и Day Close. Мета-правила для всего дня.
> **Роль:** R1 Стратег (правила) + R6 Кодировщик (исполнение сессий)

### Правила дня

| # | Правило | Суть |
|---|---------|------|
| 1 | **Слот 1 = саморазвитие** | Не переходить к текучке, пока слот саморазвития не отработан или осознанно пропущен |
| 2 | **Сессии = ОРЗ** | Каждая сессия внутри дня — полный цикл Open → Work → Close (§ Сессия ниже). Не пропускать открытие и закрытие |
| 3 | **Помидорки** | Настройки из `memory/day-rhythm-config.yaml` → `pomodoro`. По умолчанию: 25 мин работа → 5 мин перерыв. После 4 помидорок — длинный перерыв 15-30 мин |
| 4 | **Напоминание о перерыве** | Если сессия > `session_alert_minutes` (по умолчанию 50 мин) без перерыва → агент напоминает: *«Пора сделать перерыв. 5 минут.»* |
| 5 | **Сверка с планом** | Между сессиями: «Я ещё на плане дня?» Если нет — осознанное решение, не дрифт |
| 6 | **День стратегии** | Проверить `day-rhythm-config.yaml → day_open.strategy_day`. Если сегодня — не пропустить сессию стратегирования |

## § Масштаб: Сессия (Session Work)

> **Триггер:** После прохождения протокола Открытия (§ Сессия). Действует на всём протяжении стадии Работы.

## 1. Знай свои сервисы (блокирующее)

**ОБЯЗАТЕЛЬНО. ПЕРЕД началом работы над задачей.**

1. Определи, какие описания методов (сервисы) из [DP.MAP.002](../../PACK-digital-platform/pack/digital-platform/07-map/DP.MAP.002-iwe-service-catalog.md) затронуты задачей.
2. Знаешь? → Назови их (например: «Работаю с S01 Day Plan, S04 Note Review»). Переходи к п. 2.
3. Не знаешь? → **СТОП.** Сверься с MAP.002. Предложи на согласование.
4. Задача не связана ни с одним сервисом (новая фича, исследование)? → Скажи: «Новый сервис / не связано с существующими». Работай, но при Close проверь — нужно ли добавить строку в MAP.002.

## 2. Capture-to-Pack + Draft (на каждом рубеже)

> Рубеж = подзадача завершена, паттерн обнаружен, решение принято, оптимизация сделана.

На каждом рубеже спроси: **есть ли знание для записи? Есть ли зерно для поста? Есть ли идея для контента?**

| Тип знания | Куда | Когда | Через KE? |
|------------|------|-------|-----------|
| Правило для всех репо (1-3 строки) | `{{WORKSPACE_DIR}}/CLAUDE.md` | Сразу | Нет |
| Правило для одного репо (1-3 строки) | `<repo>/CLAUDE.md` | Сразу | Нет |
| Доменное (архитектура, паттерны) | Соответствующий Pack | Close | Да (KE) |
| Различение, метод, FM, WP | Соответствующий Pack | Close | Да (KE) |
| Реализационное (протоколы, процессы, конфиги) | DS docs/, PROCESSES.md, protocol-*.md | Сразу/Close | Нет |
| FAQ (доменный вопрос) | `DP.IWE.002 §11` (Pack) + README §FAQ | Сразу | Нет |
| FAQ (практический вопрос) | `LEARNING-PATH §11` (Quick Ref) | Сразу | Нет |
| Крупный урок | `memory/<topic>.md` | Close | Нет |
| Зерно для поста (связное рассуждение) | `DS-strategy/drafts/draft-list.md` + `drafts/` | Close | Нет |
| **Идея для контента** (тема + сегмент ЦА) | `DS-strategy/drafts/draft-list.md` | Сразу | Нет |

> **Все строки — captures.** В Close-отчёте указывать ВСЕ направления, не только Pack (HD #29: доменное → Pack, реализационное → DS).

> **Анонсы на рубеже:**
> - *«Capture: [что] → [куда]»*
> - *«Content idea: [тема] → [сегмент С1/С2/С3]»* — идея для публикации из текущей работы. Агрегируется в Week Review → контент-план следующей недели.
> - *«Verify: [артефакт] → /verify»* — артефакт создан и нуждается в проверке. Запускать `/verify` по запросу или отложить до Close.

## 2b. Verification on milestone (опционально + автотриггер)

> На рубеже, если создан значимый артефакт (Pack-сущность, описание метода, архитектурное решение):

**Вопрос:** *«Артефакт [X] создан. Нужна верификация? (да / на Close / нет)»*

| Ответ | Действие |
|-------|----------|
| **Да** | Запустить `/verify [артефакт]` сейчас |
| **На Close** | Добавить в список для Verification Gate (Close шаг 3b) |
| **Нет** | Пропустить (trivial / closed-loop с тестами) |

> **Не задерживать работу.** Верификация на рубеже — предложение, не блокер. Основная проверка — при Close.

### Автотриггер: после АрхГейта (S56)

> Если в сессии был АрхГейт (ЭМОГССБ-таблица, решение прошло ≥56/70) И после него менялся **код** (не только .md) → Claude **автоматически** запускает `/verify archgate` после завершения реализации.
>
> **Как отличить код от текста:** проверить расширения изменённых файлов. Код = `.py`, `.ts`, `.tsx`, `.js`, `.sh`, `.sql`, `.yaml`, `.yml`, `.json`, `.css`, `.html`. Текст = `.md`. Если только `.md` → верификация кода не нужна.
>
> **Формат анонса:** *«АрхГейт был в этой сессии. Код изменён. Запускаю Верификатора (archgate).»*

> Анонсировать: *«Capture: [что] → [куда]»*

### Творческий конвейер (4 стадии, TTL)

```
Заметка (7д) → Черновик (7д) → Заготовка (14д) → Пост
```

| Стадия | Хранение | TTL | Выходы |
|--------|----------|-----|--------|
| Заметка | `fleeting-notes.md` | 7 дней | captures / черновик / РП / archive |
| Черновик | `DS-strategy/drafts/` | 7 дней | captures / заготовка / РП / archive |
| Заготовка | `DS-Knowledge-Index` status: draft | 14 дней | пост (published) / archive |

> **Черновик ≠ Заготовка.** Черновик — личный (DS-strategy). Заготовка — публичная (DS-Knowledge-Index).
> **Заготовка ≠ Пост.** Заготовка опубликована, но не продвигается. Пост — активно продвигается.
> **Pack → пост:** KE добавил ≥3 сущностей по теме → предложить черновик для поста.
> **Guards:** ≤5 черновиков = норма. 6-10 = ⚠️ предупреждение. >10 = 🛑 блокировка (не добавлять новые).

## 2b. Видео → Captures и Контент (С4+С5)

> **Триггер:** Команда «транскрибируй видео» / `/transcribe <file>` или авто при `video.auto_transcribe.enabled: true` и длительности > `min_duration_minutes`.
> **Зависимость:** скилл `/transcribe` (MLX Whisper, Apple Silicon). Модели: `~/.local/share/mlx-whisper/mlx_models/`.

### С4: Транскрипция → Captures

1. Прочитать `memory/day-rhythm-config.yaml` → `video.auto_transcribe`
2. Транскрибировать через скилл `/transcribe <file>` (по умолчанию `distil-large-v3`, для макс. качества — `large-v3`)
3. Результат → `<video_dir>/transcripts/<filename>.txt`
4. Извлечь captures из транскрипта:
   - Упоминания РП (WP-N) → captures с привязкой к РП
   - Решения, идеи, различения → `captures.md`
   - Анонс: *«Capture (видео): [что] → [куда]»*

### С5: Видео → Посты

> **Триггер:** После С4 (есть транскрипт) + `video.content.suggest_posts: true`.

1. Проанализировать транскрипт на связные рассуждения (>3 мин на одну тему)
2. Предложить зёрна для постов:
   - *«Видео `file.mp4` содержит рассуждение о [тема] (~X мин). Предложить черновик?»*
3. Если пользователь согласен → создать черновик в `DS-strategy/drafts/` по творческому конвейеру (§2)
4. Связать черновик с исходным видео (ссылка в frontmatter)

## 2c. Skill Discovery (на рубежах)

> Рядом с Capture-to-Pack. На каждом рубеже спроси: **есть ли повторяемое действие, которое стоит оформить как скилл?**

**Критерии кандидата в скилл:**
1. Действие повторяется ≥3 раз в разных сессиях
2. Имеет чёткий вход ($ARGUMENTS) и выход (артефакт/решение)
3. Алгоритм устоялся (не меняется от сессии к сессии)

**Если все три → предложить:** *«Skill candidate: [название] — [что делает]. Оформить как `.claude/skills/[name]/SKILL.md`?»*

**Куда:** FMT-exocortex-template/.claude/skills/ (первоисточник) → sync → рабочий {{WORKSPACE_DIR}}/.claude/skills/

## 3. Pre-action Gates

| Момент | Проверка |
|--------|---------|
| Перед началом работы над задачей | Какие сервисы (MAP.002) затронуты? Нет ответа → СТОП |
| Перед `git commit` в репо с CLAUDE.md | Прочитать CLAUDE.md репо → exit protocol |
| Перед предложением архитектурного решения | АрхГейт (§5) |
| Перед commit после изменения системы | **Capture-back:** `grep memory/` на упоминания изменённой системы → обновить устаревшие записи. Анонс: *«Capture-back: [система] → memory/[файл]»* |

## 4. Общие правила работы

- **Pull-before-Commit (DS-strategy):** `git pull --rebase` → модификация → `commit` → `push`.
- **Без Obsidian (DS-strategy):** Просмотр через VS Code, НЕ Obsidian.
- **Push:** Незапушенные коммиты → ОБЯЗАН спросить «Запушить?».
