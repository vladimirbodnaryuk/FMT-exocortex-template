## SC.001: Планирование дня

**Обещание:** Начать рабочий день с ясными приоритетами, а не с пустого экрана.

**Что получаешь:**
- DayPlan — структурированный план дня
- Приоритеты из WeekPlan + carry-over с вчера
- События из Google Calendar
- Рекомендация, с чего начать

**Как это работает:** Агент-стратег автоматически формирует DayPlan к 04:00. Утром ты запускаешь Morning Check — проверяешь ночные сервисы и начинаешь работать по плану.

---

## SC.002: Планирование и ревью недели

**Обещание:** Еженедельно сверять работу со стратегией, видеть прогресс и корректировать курс.

**Что получаешь:**
- WeekPlan — план на неделю с РП, приоритетами, бюджетом
- WeekReport — итоги прошлой недели
- Возможность корректировать приоритеты и добавлять РП по ходу

**Как это работает:** В понедельник к утру готовы Session Prep (черновик плана) и Week Review (итоги). В Strategy Session ты утверждаешь план. В течение недели — Check Plan, Update Priorities, Add Workproduct.

## SC.003: Обучение и развитие

**Обещание:** Развивать компетенции через персонализированное взаимодействие с ИИ-консультантом.

**Что получаешь:**
- Ответы на вопросы по предметной области (с опорой на Pack-знания)
- Проверку домашних заданий с обратной связью по Bloom-уровням
- Ленту знаний — персонализированные материалы по расписанию
- Марафоны — пошаговые программы развития

**Как это работает:** Через Telegram-бота. Задаёшь вопрос → получаешь ответ < 3 сек. Отправляешь ДЗ → получаешь оценку. Лента и марафоны приходят по расписанию.

## SC.004: Фиксация и экстракция знаний

**Обещание:** Ни одно обнаруженное знание не теряется.

**Что получаешь:**
- Быструю фиксацию мыслей (TG → файл, < 2 мин)
- Capture-to-Pack — анонсирование знания в момент обнаружения
- Extraction Report — формализованные Pack-сущности
- Обновлённую онтологию для knowledge-mcp

**Как это работает:** Пишешь `.заметка текст` в TG — через 2 мин она в fleeting-notes.md. В рабочих сессиях Claude анонсирует Capture. Ночью экстрактор формализует captures в Pack-сущности.

## SC.005: Публикация контента

**Обещание:** Контент публикуется в клубе автоматически по расписанию.

**Что получаешь:**
- Автосканирование готовых текстов (status: ready)
- Публикация по расписанию в Discourse (systemsworld.club)
- Ручная публикация по команде /club publish
- Уведомления о новых комментариях в TG

**Как это работает:** Ставишь тексту `status: ready` → система подхватывает → публикует в запланированный слот → уведомляет о комментариях.

## SC.006: Автоматическое обслуживание

**Обещание:** Платформа работает «сама» — данные синхронизируются, шаблоны обновляются.

**Что получаешь:**
- Все ⏰-сервисы работают по расписанию без вмешательства
- Ежедневный отчёт о коммитах во всех репо
- Актуальные YAML-проекции для knowledge-mcp
- Шаблон FMT обновляется и проверяется автоматически

**Как это работает:** Scheduler Dispatch (10x/день) запускает агентов. Code Scan, Pack Projection, Template Sync — всё отрабатывает ночью. К утру — всё готово.

## SC.007: Триаж и техдолг

**Обещание:** Ни один сигнал о проблеме не теряется.

**Что получаешь:**
- Автоматическая классификация unhelpful-ответов с алертом
- Приоритизированный бэклог из feedback_triage + inbox

**Как это работает:** Пользователь ставит 👎 → Auto-Triage классифицирует → алерт в TG. По запросу — Triage Session формирует приоритизированный бэклог.

---

## SC.008: Самовосстановление

**Обещание:** Платформа сама чинит типовые проблемы — ты не замечаешь сбоев.

**Что получаешь:**
- L1: автоматический сброс зависших пользователей
- L2: исправление повторяющихся ошибок
- L3: перезапуск сервиса при критическом сбое
- L4: эскалация в GitHub Issue при нерешаемой проблеме

**Как это работает:** 4 уровня. Каждый следующий включается, если предыдущий не справился. До L4 (эскалация разработчику) — всё автоматически.

## SC.009: Аналитика и метрики

**Обещание:** Принимать решения на основе данных, а не ощущений.

**Что получаешь:**
- Ежедневный сбор метрик (качество ответов, активность, feedback)
- Отчёт по запросу (/analytics) с трендами
- Валидация Pack-сущностей по SPF
- Трекинг времени по проектам (WakaTime → Week Review)

**Как это работает:** Metrics Collection собирает данные ежедневно. /analytics — отчёт в TG. WakaTime трекает время непрерывно, данные доступны в дашборде и через API.

## SC.010: Рабочий ритм (ОРЗ)

**Обещание:** Каждый день и каждая сессия структурированы — ничего не забыто, всё зафиксировано.

**Что получаешь:**
- **День: Открытие** — DayPlan готов, календарь просмотрен, ночные сервисы проверены
- **День: Работа** — задачи выполняются в рамках РП, captures фиксируются на рубежах
- **День: Закрытие** — итоги зафиксированы, carry-over передан, backup сделан
- **Сессия: Открытие** — WP Gate пройден, РП определён
- **Сессия: Работа** — задача выполняется с Capture-to-Pack
- **Сессия: Закрытие** — KE выполнен, MEMORY обновлён, отчёт сформирован

**Как это работает:** ОРЗ — фрактальный паттерн на двух масштабах (день и сессия). Пропуск открытия = незапланированная работа. Пропуск закрытия = незафиксированный результат. Claude Code автоматически запускает соответствующие протоколы по триггерам.

## SC.011: Стратегирование

**Обещание:** Неудовлетворённости превращаются в приоритеты, а приоритеты — в утверждённый план.

**Что получаешь:**
- Strategy Session — еженедельная сессия (Пн): ревью неудовлетворённостей, приоритизация РП
- Session Prep — черновик плана, подготовленный агентом к сессии
- Week Review — итоги прошлой недели как входные данные
- Управление приоритетами по ходу недели

**Как это работает:** К утру понедельника агент готовит Session Prep (черновик WeekPlan) и Week Review (итоги). В Strategy Session ты утверждаешь план. В течение недели — Check Plan, Update Priorities, Add Workproduct по запросу.

## SC.012: Онбординг

**Обещание:** От первого контакта до первого рабочего дня — без паники.

**Что получаешь:**
- Настроенный бот — /start в TG, выбор режима, профиль заполнен
- Настроенный экзокортекс — шаблон FMT развёрнут, CLAUDE.md настроен
- Каталог сценариев — USE-CASES.md (этот файл)
- Первый рабочий день — Day Open пройден, DayPlan создан

**Как это работает:** Запускаешь setup.sh из шаблона → экзокортекс развёрнут. /start в боте → профиль настроен. Читаешь USE-CASES.md → понимаешь возможности. Первый Day Open → понимаешь ОРЗ-ритм.

---

## SC.013: Рабочая сессия с Claude Code

**Обещание:** Каждая сессия продуктивна и завершена — задача выполнена, знания зафиксированы.

**Что получаешь:**
- WP Gate — задача привязана к РП до начала работы
- UC Gate — обещание (SC) определено
- Выполненная задача — код, документ, конфигурация
- Captures — знания зафиксированы на рубежах
- Session Close — KE, MEMORY, отчёт

**Как это работает:** Даёшь задание → Claude проверяет WP Gate (есть ли РП?) → UC Gate (какой SC затронут?) → выполняет работу с Capture-to-Pack на рубежах → Close с фиксацией результатов.

## SC.014: Формализация знаний (Pack)

**Обещание:** Доменное знание не остаётся в головах — оно становится формализованными Pack-сущностями.

**Что получаешь:**
- Формализованные сущности — методы, различения, failure modes, SOTA, рабочие продукты
- Валидация по SPF — каждая сущность проверена на соответствие формату
- Связи через frontmatter — доступны через MAP.001
- Обновлённая онтология — knowledge-mcp получает новые сущности

**Как это работает:** Captures из рабочих сессий → Knowledge Extraction формализует в Pack-сущности (ID, frontmatter, status) → Ontology Sync обновляет knowledge-mcp → сущности доступны боту и агентам.

## SC.015: Развитие системы (DS)

**Обещание:** Любое изменение проходит полный цикл — от обещания до работающего сервиса.

**Что получаешь:**
- UC Gate пройден — обещание (SC) определено до начала
- IntegrationGate — контур, роли, продукты, процессы определены
- АрхГейт ≥8 — решение оценено по ЭМОГССБ
- Работающий сервис — код, PROCESSES.md, строка в MAP.002
- Тест на pilot — проверено до мержа в прод

**Как это работает:** РП из WeekPlan → UC Gate (какой SC?) → IntegrationGate (контур?) → АрхГейт (ЭМОГССБ ≥8) → реализация на pilot → тест → cherry-pick на прод → MAP.002 + PROCESSES.md обновлены.
