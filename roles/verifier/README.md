# Верификатор (R23)

> **Модуль шаблона:** `roles/verifier/` в [FMT-exocortex-template](../../README.md)
> **Роль:** R23 Верификатор — проверка артефактов по эталону (PACK-verification VR.R.001, DP.ROLE.001)

Роль Верификатор проверяет соответствие артефактов эталону (Pack, SPF, принципы). Ключевой принцип — **context isolation**: верификатор загружает другой контекст, чем создатель.

---

## Когда вызывается

| Режим | Триггер | Как |
|-------|---------|-----|
| **По запросу** | Явная команда пользователя | `/verify [артефакт]` |
| **Quick Close** | Верификация чеклиста (шаг 7 protocol-close § Сессия) | Sub-agent Haiku, context isolation |
| **Day Close** | Верификация чеклиста (шаг 10 protocol-close § День) | Sub-agent Haiku, context isolation |
| **Session Close** | Verification Gate (шаг 3 protocol-close) | Автоматически для open-loop/problem-framing РП |
| **На рубеже** | Создан значимый артефакт | Предложение в protocol-work §2b |

## Что проверяет

| Тип | Глубина | Что |
|-----|---------|-----|
| **Формальная** (VT.001) | Структура, frontmatter, ссылки | Низкая — автоматизируема |
| **Содержательная** (VT.002) | Принципы, UL, непротиворечивость | Средняя — требует Pack-контекст |
| **Кросс-контекстная** (VT.003) | Pack↔DS, BC↔BC согласованность | Высокая — несколько источников |

## Context isolation

```
Верификатор загружает:          Верификатор НЕ загружает:
├─ Артефакт (результат)         ├─ Задание создателя
├─ Эталон (Pack/SPF)            ├─ Промежуточные рассуждения
└─ Чеклист (если есть)          └─ Историю итераций
```
