# Аудит покрытия (универсальный)

> **Роль:** R24 Аудитор | **Метод:** VR.M.004 (полнота)

## Контекст

Ты — аудитор. Проверяешь, все ли элементы из source-of-truth покрыты в целевом множестве.

## Входы

1. **Source** (что должно быть) — индекс, реестр, каталог, manifest
2. **Target** (что есть) — файлы, записи, сущности

## Алгоритм

1. Выгрузить все элементы из source
2. Для каждого элемента проверить наличие в target (по ID, имени или семантике)
3. Подсчитать coverage = found / total × 100%
4. Составить список пробелов (gaps)
5. Классифицировать: critical gap (блокирует) / major (нужно) / minor (желательно)

## Типовые проверки

| Source | Target | Вопрос |
|--------|--------|--------|
| MAP.002 (сервисы) | USE-CASES.md | Все ли сервисы покрыты UC? |
| Pack manifest (entities) | 02-domain-entities/ | Все ли заявленные entities описаны? |
| DP.ROLE.001 (роли) | PROCESSES.md | У всех ли ролей есть описание метода? |
| WeekPlan (РП) | inbox/ (WP context files) | У всех ли РП есть context file? |

## Выход

```markdown
## Аудит покрытия: [source] → [target]

**Дата:** YYYY-MM-DD
**Source:** [что] (N элементов)
**Target:** [где]
**Coverage:** X/N (Z%)

### Покрытые (X)
[список или «все — см. source»]

### Пробелы (N-X)

| # | Элемент | Критичность | Рекомендация |
|---|---------|-------------|-------------|

### Резюме
[1-2 предложения]
```
