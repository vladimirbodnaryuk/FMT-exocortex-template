# Аудитор (R24)

> **Модуль шаблона:** `roles/auditor/` в [FMT-exocortex-template](../../README.md)
> **Роль:** R24 Аудитор — проверка полноты и согласованности (PACK-verification VR.R.002, DP.ROLE.001)

Роль Аудитор систематически проверяет полноту и покрытие: все ли элементы на месте, нет ли расхождений между связанными контекстами.

---

## Когда вызывается

| Режим | Триггер | Как |
|-------|---------|-----|
| **Day Open** | Проверка согласованности планов | Автоматически (шаг 2 protocol-open) |
| **Стратегирование** | Полный аудит РП и покрытия | Автоматически |
| **По запросу** | «Проверь покрытие MAP.002 сценариями» | Явная команда |

## Два метода

### VR.M.002 — Кросс-контекстная проверка

Ищет расхождения между связанными контекстами:

| Паттерн | Описание |
|---------|----------|
| **Orphan** | Есть в A, нет в B |
| **Ghost** | Есть в B, нет в A |
| **Drift** | Есть в обоих, но расходятся |
| **Stale** | Один обновлён, другой — нет |

### VR.M.004 — Аудит полноты

Проверяет coverage: все ли элементы source покрыты в target.

## Промпты

| Промпт | Сценарий |
|--------|----------|
| `audit-plan-consistency.md` | MEMORY↔WeekPlan↔DayPlan↔Registry |
| `audit-coverage.md` | Любой source×target (MAP.002↔UC, manifest↔entities) |

## Рабочие продукты

- **Отчёт аудита** (VR.WP.003): coverage %, список пробелов
- **Список несоответствий** (VR.WP.004): расхождения между контекстами
