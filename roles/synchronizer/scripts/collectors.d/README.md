# collectors.d/ — плагины dt-collect.sh

Персональные коллекторы метрик для ЦД. Каждый файл `*.sh` автоматически загружается при запуске `dt-collect.sh`.

## Формат плагина

```bash
#!/bin/bash
# COLLECTOR: имя_функции
# TARGET: 2_7_iwe | 2_8_ecosystem | 2_9_knowledge
# Описание: что собирает этот коллектор

collect_имя_функции() {
    # Доступны: $WORKSPACE, log()
    # Должен вывести JSON в stdout
    # При ошибке или отсутствии данных — вывести {}
    echo '{"my_metric": 42}'
}
```

## Обязательные поля

- `# COLLECTOR:` — имя функции (без `collect_` префикса)
- `# TARGET:` — секция JSONB в `digital_twins.data → 2_collected`:
  - `2_7_iwe` — метрики IWE (РП, мультипликатор, scheduler)
  - `2_8_ecosystem` — метрики экосистемы (бот, публикации)
  - `2_9_knowledge` — метрики знаний (Pack, Scout, заметки)

## Правила

1. Плагин **должен** возвращать `{}` если данных нет (файл не найден, сервис не настроен)
2. Плагин **не должен** падать с ошибкой — stderr перенаправляется в лог
3. Имена метрик — уникальные в пределах TARGET-секции
4. `update.sh` **не затрагивает** эту директорию — ваши плагины сохраняются при обновлении шаблона

## Примеры

Персональные коллекторы (L4) — для систем, которые есть не у всех:

- `scout.sh` — ночной разведчик (если настроен DS-agent-workspace/scout/)
- `unsatisfied.sh` — QA бота (если настроен DS-agent-workspace/scheduler/feedback-triage/)
- `publications.sh` — публикации (если есть DS-Knowledge-Index-*/docs/)
