# Повестка стратегической сессии

> Шаблон. Используется в `strategy_day` (по умолчанию — понедельник).
> Source-of-truth алгоритма: `DP.ROLE.012.SC.01` (Pack-сценарий Стратега).

---

## Типовая сессия (~1.5–2ч)

### 1. Ревью НЭП (5–10 мин)
- [ ] Загрузить `docs/Dissatisfactions.md`
- [ ] Появились новые неудовлетворённости? Изменились приоритеты?
- [ ] Если да → обновить, отразить связь НЭП → Strategy.md
- [ ] Проверить операционные НЭП: прогресс?

### 2. Анализ прошлой недели (15–20 мин)
- [ ] Загрузить WeekPlan прошлой недели
- [ ] Собрать коммиты за неделю из **ВСЕХ** репо
- [ ] Сопоставить РП и коммиты
- [ ] Рассчитать `completion_rate = done_wp / planned_wp * 100%`

### 3. Сдвиг фокуса месяца (10–15 мин)
- [ ] Загрузить `docs/Strategy.md` → секция «Фокус: [месяц]»
- [ ] Обновить ревью-таблицу за прошедшую неделю
- [ ] Проверить ТОС-месяца: ограничение снято?
- [ ] Проверить гипотезу: подтверждается?

### 4. Формирование плана недели (20–30 мин)
- [ ] Выбрать РП из результатов месяца + carry-over
- [ ] Для каждого РП указать: к какому результату месяца он ведёт
- [ ] Распределить бюджет времени
- [ ] Определить ТОС-недели (узкое горлышко)
- [ ] Сформировать WeekPlan

### 5. Запрос к пользователю
- [ ] Показать итоги прошлой недели
- [ ] Показать ТОС-недели и план (таблица РП)
- [ ] Запрос на корректировку

### 6. Фиксация (после подтверждения)
- [ ] Сохранить WeekPlan
- [ ] Обновить Strategy.md
- [ ] Обновить MEMORY.md → «РП текущей недели»
- [ ] Закоммитить изменения

---

## Месячная надстройка (~30 мин дополнительно)

> Выполняется на **первой** стратегической сессии месяца.

- [ ] Ретро месяца: результаты vs факт (done / carry-over / отменено)
- [ ] Архивация текущего фокуса месяца
- [ ] Новый фокус месяца (состояние, ТОС, гипотеза, результаты, маппинг РП)
- [ ] Обновить Dissatisfactions.md
- [ ] Обновить цели квартала
